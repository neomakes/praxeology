# Velocity Labs Mission

# Article 1 — Reason for Existence

1. Velocity Labs builds developer tooling that removes friction from the software delivery cycle.
2. The core belief: every hour a developer spends on tooling is an hour not spent on product.
3. We exist to give that time back.

# Article 2 — Core Values

1. Ship fast, fix faster: speed is a feature; so is reliability. Both are required.
2. Developer empathy: we are the user. We build tools we would use ourselves.
3. Ruthless simplicity: the best feature is the one that does not need to exist.
4. Evidence over opinion: decisions are driven by data, user feedback, and measurable outcomes.

# Article 3 — North Star

1. Reach 10,000 active developer users within 18 months.
2. Every release improves at least one measurable developer workflow metric.
3. The product must be runnable by a solo developer in under 5 minutes.

# Article 4 — Product Principles

1. Default to CLI-first. GUIs are optional enhancements.
2. Configuration is code. No magic environment-specific behavior.
3. Errors are human-readable. Stack traces are for logs, not users.
4. Backwards compatibility is sacred once a feature is stable.

# Article 5 — Team Operating Model

1. Founder (Alex) owns product direction and final technical decisions.
2. Agents operate within defined domains and escalate blockers within 15 minutes.
3. The backlog is the source of truth for what to build next — not ad hoc requests.
4. Shipping a working v1 beats planning a perfect v2.
